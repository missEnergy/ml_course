def loss(X, y_one_hot, Thetas, llambda):
    y = y_one_hot
    total_loss = 0
    for i in range(len(y[0])):
        total_loss = total_loss + (-y[:, i].dot(np.log(predict_vector(X, Thetas)[:, i]))-(1-y[:, i]).dot(np.log(1-predict_vector(X, Thetas)[:, i])))/len(X)
    for i in range(len(Thetas)):
        relevant_thetas = Thetas[i][:, 1:].flatten()
        total_loss = total_loss + llambda/2/len(X)*relevant_thetas.dot(relevant_thetas)
    return total_loss

1. How does back propagation actually work?¶

Okay, so I found out back propagation is actually just 1 step in the algorithm.

Actually my question is, how does a neural net train it's weights exactly?

Turns out that's actually not that hard to understand!

Step 1: initialize weights and weight-updates (= gradients)¶

def initialize_weights(input_size, hidden_layer_size, output_size):
    Thetas = []
    Thetas.append(np.random.uniform(-0.12, 0.12, [hidden_layer_size, input_size + 1]))
    Thetas.append(np.random.uniform(-0.12, 0.12, [output_size, hidden_layer_size + 1]))
    return Thetas

def initialize_gradients(input_size, hidden_layer_size, output_size):
    Gradients = []
    Gradients.append(np.zeros([hidden_layer_size, input_size + 1]))
    Gradients.append(np.zeros([output_size, hidden_layer_size + 1]))
    return Gradients

Step 2: for each x in pseudo code¶

  • compute the activation value of each node by a feed forward through the net
  • determine how much each node contributed to the loss of the prediction (=deltas). This is actually the back propagation step!
  • determine how much the weights should be updated to get a smaller loss

Step 2: for each x in actual code¶

In [12]:
def get_activations_by_single_feed_forward(x, Thetas):
    activations = []
    activations.append(np.concatenate(([1],x))) # add intercept
    for i in range(len(Thetas)):
        next_activations = sigmoid(activations[i].dot(np.transpose(Thetas[i])))
        if i == len(Thetas) - 1:
            activations.append(next_activations)
        else:
            activations.append(np.concatenate(([1],next_activations))) # add intercept
    return activations

def get_deltas_by_back_prop(y, activations, Thetas):
    deltas = []
    deltas.append(activations[-1] - y)
    for i in reversed(range(1, len(Thetas))):
        previous_deltas = np.transpose(Thetas[i]).dot(deltas[0])*activations[i]*(1-activations[i])
        deltas.insert(0, previous_deltas)
    return deltas

def accumulate_Gradients(Gradients, activations, deltas):
    for i in range(len(Gradients)):
        if i == len(Gradients) - 1:
            Gradients[i] = Gradients[i] + np.outer(deltas[i], activations[i])
        else:
            Gradients[i] = Gradients[i] + np.outer(deltas[i][1:], activations[i])
    return Gradients

Step 3: update the weights!¶

In [13]:
def regularlize_Gradients(Gradients, llambda, Thetas, X):
    for i in range(len(Gradients)):
        Gradients[i] = Gradients[i] / len(X) + np.c_[ np.zeros(Thetas[i].shape[0]), Thetas[i][:, 1:] ] * llambda / len(X)
    return Gradients

def gradient_descent(Thetas, Gradients, alfa):
    for i in range(len(Thetas)):
        Thetas[i] = Thetas[i] - alfa * Gradients[i]
    return Thetas

2. How can a neural net approximate complex features?¶

  • Neural nets can approximate any function
  • We don't need to feature engineering - the net will do that for us
  • So how can it find relevant features with only linear combinations and activations?

More explicitly how can it approximate

  • logical operators,
  • polynomial terms (e.g. $x_1^ 2$ or $x_1 * x_2$)
  • min/max

Logical operators¶

Polynomial terms (e.g. $x_1^ 2$ or $x_1 * x_2$)¶

This blogpost explains that:

  • applying log() turns multiplications into additions, job done! This is also useful to limit the range of input values.
  • secondly, neural networks can approximate arbitrary functions. And of course, it can approximate a multiplier as wells
  • however, the model just learns to approximate the output for the examples in the training data, it understands nothing about multiplication. In other words, what the model has learnt doesnt generalize to true multiplication.
  • so if you have a strong intuition that multiplication is the right way to model certain relations, then it may be better to explicitly enforce that in the network.
